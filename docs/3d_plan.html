Octahedron Encoding

// Encode float2 oct_encode(float3 n) { n = normalize(n); float2 oct = n.xy * (float)(1.0 / (abs(n.x) + abs(n.y) + abs(n.z))); return oct * 0.5 + 0.5; } // Decode float3 oct_decode(float2 oct) { oct = oct * 2.0 - 1.0; float3 n; n.z = 1.0 - abs(oct.x) - abs(oct.y); if (n.z >= 0.0) { n.x = oct.x; n.y = oct.y; } else { float2 f2 = float2(1.0 - abs(oct.y), 1.0 - abs(oct.x)) * sign(oct); n.x = f2.x; n.y = f2.y; } return normalize(n); }

4. Cascaded Shadow Maps (CSM)

參數值
Cascade 數量3
分割策略對數分割（Logarithmic）
Cascade 02048×2048, near → ~4.6m
Cascade 11024×1024, ~4.6m → ~21.5m
Cascade 2512×512, ~21.5m → 100m
Filtering4-tap PCF
Max Directional Lights4

Logarithmic Split

split_i(near, far, i, N) = near * (far / near)^(i / N) Cascade 0: 0.1m → ~4.6m Cascade 1: ~4.6m → ~21.5m Cascade 2: ~21.5m → 100m

Single-Pass CSM（Viewport Scissors）

Shadow Atlas (2048×2048): ┌─────────────┬─────────────┐ │ Cascade 0 │ Cascade 1 │ │ 2048² │ 1024² │ ├─────────────┼─────────────┤ │ Cascade 2 │ │ │ 512² │ free │ └─────────────┴─────────────┘

4-tap PCF

float shadow_pcf(sampler2D shadow_map, float4 sc, float bias, float2 size) { float2 ts = 1.0 / size; float s = 0.0; s += texture(shadow_map, sc.xy + vec2(-ts.x,-ts.y)).r; s += texture(shadow_map, sc.xy + vec2( ts.x,-ts.y)).r; s += texture(shadow_map, sc.xy + vec2(-ts.x, ts.y)).r; s += texture(shadow_map, sc.xy + vec2( ts.x, ts.y)).r; return (s / 4.0) < sc.z - bias ? 0.0 : 1.0; }

衰減公式

Point Light

att = (1 - dist/range)²

Spot Light

spotEffect = dot(L, -spotDir) angleAtt = smoothstep(outer, inner, spotEffect) totalAtt = (1 - dist/range)² × angleAtt

Tiled / Clustered Lighting

ComputeShader 把屏幕分成 16×16 tiles，每個 tile 預先計算影響它的燈光（平均 2-4 個）。省 50-80% 燈光遍歷計算。

5b. PBR 環境光

G-buffer 只儲存 material properties（normal, albedo, roughness, metallic, ao），環境光需要另外處理。

三層環境光

層次內容計算方式成本
Ambient簡單常數或天空顏色albedo × ambient × ao極低
SH Irradiance球諧函數預計算的間接光albedo × SH_probe(pos, N)低
IBL ReflectionEnvironment map 反射env_map.sample(reflection_vec, roughness)中

Phase 1：Unity 風格預設天空（已實現）

float3 GetSkyAmbient(float3 N, float3 albedo, float ao) { float t = N.y * 0.5 + 0.5; // -1~1 → 0~1 float3 sky_color = float3(0.5, 0.7, 1.0); // 藍天 float3 ground_color = float3(0.2, 0.2, 0.2); // 灰色地 float3 sky_ambient = lerp(ground_color, sky_color, t); return sky_ambient * albedo * ao * ambient_strength; }

SH Irradiance（Phase 2+）

struct SHIrradiance { float3 L00, L1m1, L10, L11; // Band 0 + 1 float3 L2m2, L2m1, L20, L21, L22; // Band 2 }; float3 SampleSH(SHIrradiance sh, float3 N) { return sh.L00 + sh.L1m1*(N.y) + sh.L10*(N.z) + sh.L11*(N.x) + sh.L20*(N.y*N.z) + sh.L21*(N.y*N.x) + sh.L22*(N.z*N.x) + sh.L2m2*(N.x*N.y) + sh.L2m1*(N.z*N.z - 1.0/3.0); }

IBL Reflection（Phase 3+）

// Specular IBL float3 R = reflect(-V, N); float mip = roughness * (num_mips - 1); float3 specular = env_cubemap.SampleLevel(R, mip); // PBR BRDF float3 F0 = lerp(float3(0.04), albedo, metallic); float3 F = FresnelSchlickRoughness(NdotV, F0, roughness); float3 kD = (1 - F) * (1 - metallic); float3 ambient = kD * diffuse_sh + specular * F;

SH Light Probe（Baked GI，Phase 2）

Light Probe 解決物件之間的全局光照（light bounce）。Editor 中放置 probes，烘焙 SH irradiance 係數。

Probe 放置

場景類型放置策略
室外每 10m 放置一個
室內每個房間 4-8 個，角落更密集

Editor 烘焙流程

// 1. 在每個 probe 位置發射 rays，採樣周圍環境 // 2. 對採樣結果做 SH projection，得到 9 個係數 // 3. 儲存到 comp_3d_environment_probe.sh struct SHIrradiance { float3 L00, L1m1, L10, L11; // Band 0+1 float3 L2m2, L2m1, L20, L21, L22; // Band 2 };

運行時使用

// 根據 world position 找最近的 probe comp_3d_environment_probe* probe = find_nearest_probe(world_pos); // 計算該方向的 irradiance float3 indirect_light = SampleSH(probe->sh, normal); // 應用於 PBR float3 diffuse_indirect = albedo * indirect_light * ao; color = direct_lighting + diffuse_indirect;

多 Probe 混合

// 找最近的 2-4 個 probes，根據 distance 混合 float w0 = 1.0 - smoothstep(0, blend_radius, dist0); float w1 = 1.0 - smoothstep(0, blend_radius, dist1); float3 final_sh = (sh0 * w0 + sh1 * w1) / (w0 + w1);

8. Forward+ / Deferred 雙 Pipeline 支援

Forward+ vs Deferred 比較

特性Forward+Deferred
原理Tiled Forward + Light BinningG-buffer + Lighting Pass
燈光數量中等（~64/tile）多（所有燈光一次處理）
透明物件原生支援（Alpha Blend）需要 Dual-Render
適用室內、少量動態燈光室外、大量燈光

Forward+ 原理

Pass 1: Depth Pre-pass Pass 2: Light Binning (ComputeShader, 16×16 tiles) Pass 3: Forward Tiled Rendering (每個 tile 平均 2-4 燈光)

架構

PipelineManager ├── Forward+ Path │ Depth → Tiled Light Bin → Tiled Forward → screen └── Deferred Path G-buffer → Lighting → PostFX → screen

9. 效能優化

Early-Z + Depth Pre-pass P0
Depth pre-pass → G-buffer pass 開啟 Early-Z，自動 reject 被遮擋 fragment。省 30-60% fragment shader。
G-Buffer 降精度 P0
RGBA64 → RGBA16F, D32 → D24S8。省 25-50% 頻寬，精度足夠 PBR。
Single-Pass CSM P0
Viewport scissors 一次 draw call 輸出 3 個 cascade。省 2/3 shadow draw calls。
DrawProceduralIndirect P0
GPU-driven draw call，減少 CPU draw call overhead。
Tiled Lighting P1
Screen-space tile light binning，每個 tile 平均 2-4 個燈光。省 50-80% lighting 計算。
Hi-Z Occlusion P1
GPU CS 對 visible list 中的大物件做 depth mipmap 查詢，進一步剔除被遮擋物件。
Variable Rate Shading P2
周边区域像素著色降低，進一步優化效能。

11. 實施階段

Phase 1: 基礎建設
  • G-Buffer 渲染（gbuffer0, gbuffer1, main depth）
  • Directional Light 基本 lighting（無陰影）
  • Basic camera + mesh rendering
  • 基礎 Frustum Culling（CPU 版本）
Phase 2: 完整陰影
  • CSM for Directional Light（3 cascades, Single-Pass）
  • Point Light + Cubemap Shadow
  • Spot Light + Perspective Shadow Map
  • 完整 3 種燈光 + 陰影
Phase 3: 高級渲染
  • Post-Processing（SSAO, Bloom, TAA）
  • Tiled / Clustered Lighting
  • Hi-Z Occlusion Culling
  • LOD System
  • 透明物件 Transparent Forward Pass
  • 粒子系統 + Additive Glow + Bloom
Phase 4: 優化整合
  • Early-Z + Depth Pre-pass
  • G-Buffer 降精度（RGBA16F + D24S8）
  • DrawProceduralIndirect
  • Variable Rate Shading
