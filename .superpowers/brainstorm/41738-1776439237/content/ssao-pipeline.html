SSAO 管線流程與潛在問題

深入分析 engine SSAO 顯示效果問題

Pass 3a: SSAO (半分辨率)

// postfx_ssao.kong
1. 讀取 depth_target (全分辨率深度)
2. 線性化深度 + 16方向採樣
3. 輸出到 ao_result @ width/2 × height/2
ao_result 格式: RGBA64
默認參數: radius=0.5, strength=0.8, near=0.1, far=100.0

Pass 4: Compositor (全分辨率)

// postfx_compositor.kong
1. 讀取 gbuffer0 (全分辨率)
2. 讀取 ao_result @ width/2 × height/2
3. 問題: 使用全分辨率UV採樣半分辨率紋理
4. color * ao → 輸出到屏幕

🔴 核心問題: 分辨率不匹配

ao_result 是半分辨率 (W/2 × H/2)，但 compositor 用全分辨率 UV 採樣

Compositor 紋理坐標范圍 [0,1]，但 ao_result 實際隻有 [0,0.5] × [0,0.5] 的有效區域

→ 採樣坐標嚴重重疊，AO 效果混亂

其他可能問題

可能原因

  • 分辨率不匹配 — 最可能，AAO紋理被壓縮到1/4區域
  • 深度值範圍 — near/far 設置與實際場景不匹配
  • Blend 模式 — pipeline blend 設置可能不正確
  • Clear color — 0xffffffff 可能影響初始深度

不太可能

  • Shader 邏輯 — 16採樣算法本身看起來合理
  • 紋理格式 — RGBA64 足夠存儲單通道 AO
