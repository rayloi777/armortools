粒子 + Bloom 解決方案

Additive Particles 天然支援 Bloom

關鍵：粒子發光在 HDR buffer 裡

粒子 Additive Blend 的本質： result = buf_color + particle_color 粒子 Bloom 的本質： bloom = threshold_extract(buf_color) → blur → add_back 結論：粒子用 Additive blend 寫入 HDR buffer (buf/last)， bloom threshold 會自動偵測到粒子高亮度值並做 bloom！

粒子 Blend Mode 選擇

Blend Mode公式適用場景支持 Bloom？
Additivebuf + particle火焰、爆炸、光環、魔法✅ 天然支持
Alpha Blendbuf × (1-α) + p × α煙霧、雲朵、半透明物體⚠️ 低亮度難觸發
Soft Addbuf + p × α柔光效果✅ 支持
Multiplybuf × particle陰影、著色❌ 無法 bloom

粒子 Component 設計

Tag: CompParticle // 粒子物件 Tag: CompGlowingParticle // 發光粒子（影響 bloom） typedef struct { float alpha; // 透明度 float emissive_strength; // 發光強度（高於 1.0 才會觸發 bloom threshold） float3 color; // 粒子顏色 int blend_mode; // 0=Additive, 1=Alpha, 2=SoftAdd, 3=Multiply } comp_particle; // Bloom threshold: 通常設為 0.8-1.0 // 所以 emissive_strength 需要 > threshold 才會被 bloom 吃到

粒子系統整合（Minic）

// Minic API id particle_system_create(int max_count); void particle_emit(id system, float3 pos, float3 vel, float life); void particle_set_glow(id system, float strength, float3 color); void particle_set_blend(id system, int blend_mode); // 0=Additive // 使用範例 id fire = particle_system_create(1024); particle_set_blend(fire, 0); // Additive particle_set_glow(fire, 2.5, float3(1.0, 0.4, 0.1)); // 強度 2.5 > bloom threshold 1.0

Post-FX 順序（非常重要）

SSAO → Lighting → Particles(Additive) → Bloom → TAA → Compositor 順序關鍵： 1. SSAO 在 lighting 之後：AO 影響不透明物體 2. Particles 在 Bloom 之前：粒子 HDR 值必須先寫入 buf 3. Bloom 順序： buf → BrightnessExtract → Blur → Composite → last 4. TAA 在 Bloom 之後：TAA 需要乾淨的 frame

粒子需要陰影嗎？

一般粒子（火焰、爆炸）不投射陰影，節省效能。

特殊情況（如體積雲、濃密煙霧）可用 Light Shaft 或 Volumetric Light 模擬。
