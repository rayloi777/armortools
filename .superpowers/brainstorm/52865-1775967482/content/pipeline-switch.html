Forward+ / Deferred 雙 Pipeline 支援

統一 Material 接口 + Runtime Pipeline 切換

問題

用家需求

  • 場景 A 用 Deferred（多燈光）
  • 場景 B 用 Forward+（少量燈光，快速）
  • 同一遊戲中不同關卡切換
  • 手機上用 Forward+，PC 上用 Deferred

挑戰

  • Deferred 需要 G-buffer，Forward+ 不需要
  • Shader 必須同時支援兩種 pipeline
  • Material 接口要統一

解決方案：Pipeline 抽象層

核心思想： 1. 統一的 Material 接口（albedo, normal, roughness, metallic...） 2. PipelineManager 根據當前設定選擇： - Forward+ path: depth → tiled light bin → forward_tiled_lighting → screen - Deferred path: gbuffer → lighting → screen 3. Shader 自動適配（同一個 shader 可以在兩種 pipeline 下工作）

Forward+ vs Deferred 比較

特性Forward+Deferred
原理Tiled Forward + Light BinningG-buffer + Lighting Pass
燈光數量中等（~64 燈光/tile）多（所有燈光一次處理）
透明物件原生支援（Alpha Blend）需要 Dual-Render
硬體需求中等較高（G-buffer bandwidth）
Post-FX簡單複雜
適用場景室內、少量動態燈光室外、大量燈光

Forward+ 原理

Pass 1: Depth Pre-pass（可選 Early-Z） Pass 2: Light Binning（ComputeShader） 把屏幕分成 16×16 tiles 每個 tile 計算影響它的燈光列表 Pass 3: Forward Tiled Rendering 每個 fragment 先確定自己屬於哪個 tile 只遍歷該 tile 的燈光列表（平均 2-4 個）

PipelineManager

typedef enum { PIPELINE_FORWARD_PLUS, PIPELINE_DEFERRED, } pipeline_type; void pipeline_set(pipeline_type type) { switch (type) { case PIPELINE_FORWARD_PLUS: render_path_commands = sys_3d_render_commands_forward_plus; break; case PIPELINE_DEFERRED: render_path_commands = sys_3d_render_commands_deferred; break; } }

Quality Preset（方便用家）

// Minic API void render_set_quality(const char* preset); enum quality_preset { QUALITY_LOW, // Forward+, no shadows, no post-fx QUALITY_MEDIUM, // Forward+, basic shadows QUALITY_HIGH, // Deferred, CSM, SSAO QUALITY_ULTRA, // Deferred, CSM×4, SSAO, Bloom, TAA };

平台自動選擇

void pipeline_auto_select(void) { if (gpu_is_mobile()) { pipeline_set(PIPELINE_FORWARD_PLUS); // 省 bandwidth } else { pipeline_set(PIPELINE_DEFERRED); // PC/Console } }
