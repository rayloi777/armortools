全局光照（GI）— 物件之間互相照射

Baked GI · Real-time GI · 混合方案

問題

場景中物件互相照射： - 紅色地板反彈紅光到牆壁 - 白色天花板反彈白光到所有物件 - 窗戶外陽光反射到室內 Direct Lighting: 只計算燈光直射 SH Irradiance: 環境光照，不能表達物件之間的互相照射 所以場景會看起來很"平"，缺乏真實感。

三種解決方案

1. Baked GI（烘焙 GI）

Editor 預計算 lightmap / light probe。運行時直接使用。

  • Unity Enlighten, Unreal LPV
  • 最高質量，靜態場景必備
  • 缺點：只適合靜態物件
Production Ready

2. Real-time GI（即時 GI）

運行時即時計算。可以處理動態物件。

  • Ray Tracing GI (DXR, RTX)
  • Voxel GI (UE5 Nanite Lumen)
  • 缺點：硬體要求高
High Cost

3. Screen-space GI

利用 depth buffer 和 normal buffer 估算 GI。便宜的近似。

  • SSAO（只算遮擋）
  • SSDO（計算方向性反彈）
  • 缺點：只看得到螢幕上的像素
Approximation

1. Baked GI — Lightmap + Light Probe

Lightmap

每個 static 靜態物件的表面 UV 展開， 烘焙時 ray trace 計算每個 texel 的光照： texel(u,v) = direct_light + Σ(indirect_light from all directions) 運行時：texture sampling 即可 優點：超高質量 缺點：需要 UV 展開，靜態物件

Light Probe

場景中均勻放置 probe： 每個 probe 記錄 SH irradiance 係數（27 floats） runtime 時： nearest_probes = find_N_nearest_probes(world_pos) color = interpolate_SH(probes, N) 動態物件經過時，採樣最近的 probe

2. Real-time GI — Voxel GI + Ray Trace

Voxel GI（Lumen / LPV）

1. 把場景 voxelize 成 3D grid 2. 用 SH 表示每個 voxel 的光照 3. 光線傳播（Light Propagation）： 每幀把光照在 voxel grid 中傳播 4. 渲染時查詢 fragment 位置的 voxel GI UE5 Lumen 用 Hardware Ray Trace + Voxel Hybrid

Ray Tracing GI

每個 fragment 發射 1-4 條 ray： ray_bounce_0 = direct lighting ray_bounce_1 = trace(pos, random_dir) // 第一次反彈 ray_bounce_2 = trace(pos, random_dir) // 第二次反彈 多次 bounce 的結果混合： color = direct + 0.5*indirect_1 + 0.2*indirect_2 DXR / Metal Ray Tracing 硬體支援

3. Screen-space GI — SSDO

利用 G-buffer（depth + normal）估算 indirect lighting： for each pixel p: for each sample s in hemisphere(N, p): if depth(s) < depth(p) + threshold: // s 在 p 的"遮擋物後面" indirect += light_from_direction(s - p) * ao_weight else: // s 是可見的天空/物件 indirect += sample_gbuffer_normal(s) * visibility(s,p) SSDO = SSAO + 方向性 light bounce 比 SSAO 貴，但能得到 light bounce 效果

Engine 建議方案

1
Phase 1-2: SH Light Probe 烘焙。Editor 中放置 probes，烘焙 SH irradiance。動態物件採樣最近 probe。
2
Phase 3: 加入 SSDO（基於現有 SSAO pass）。便宜能得到 light bounce 效果。
3
Phase 4+: 如果 GPU 支援，加入 Voxel GI 或 Ray Trace GI。

Light Probe 放置

室外場景： 每 10m 放置一個 probe（基於場景大小調整） 只需要 1 套 SH coefficients 室內場景： 每個房間 4-8 個 probes（角落、光源附近更密集） 每個 probe 的 SH 代表"房間的整體光照" 創意放置： - 光源附近放置更密集 - 角落（light bounce 多的地方）放置更密集
