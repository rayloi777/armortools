效能優化建議

點擊各項目查看詳細說明

A

1. Single-Pass CSM（Viewport Scissors）

3 個 cascade 不要分 3 次渲染，用 scissor test 在一次 draw call 中畫到 3 張不同的 shadow map

  • 減少 66% shadow draw calls
  • Geometry shader 一次性輸出到 multiple targets
  • 或用 Multi-draw indirect + scissor

省 2/3 shadow draw calls

B

2. Tiled / Clustered Lighting

不對每個 fragment 遍歷所有燈光，而是把屏幕分成 16×16 tiles，每個 tile 預先計算哪些燈光影響它

  • 減少 8× point lights 的無效 lighting pass 採樣
  • 每個 tile 只計算 1-4 個影響它的燈光
  • 可加進 deferred lighting pass 前做

省 50-80% lighting 計算量

C

3. Early-Z / Depth Rejection

Geometry pass 前先做深度 pre-pass，把最新深度寫入 depth buffer，後續 fragment shader 可 skip 被遮擋的像素

  • Z-cull 硬體自動做
  • 需要 G-buffer pass 也開啟 depth test (equal/less)
  • 光柵化階段就 reject 被遮擋的 fragment

省 30-60% fragment shader 开销

D

4. G-Buffer 壓縮 / 降精度

RGBA64 (64bit) → RGBA16F (64bit) 或 RGBA8 (32bit) — 取決於 GPU 精度需求

  • Normal octahedron 只需 2 通道，RGBA8 足夠
  • Depth 用 D24S8 而非 D32
  • 省 50% gbuffer 記憶體頻寬

省 50% gbuffer 頻寬
