Frustum Culling 方案

點擊方案查看說明

A

Frustum Culling（視錐裁剪）

對每個物件的包圍盒（AABB / OBB）做 6 個 frustum plane 檢測

  • 最基礎、必做
  • CPU 端每幀檢測所有物件
  • 簡單場景可忽視，1000+ 物件時必須

所有方案的核心基礎

B

Clustered Frustum Culling

先把場景分成 Grid / Octree / BVH 叢集，對叢集做 frustum test

  • 大量小物件時大幅減少測試次數
  • Octree 適合靜態場景
  • Grid 適合大室外場景

適合大型開放世界

C

Hi-Z（Hierarchical Z-Buffer）Culling

利用前一幀的 depth buffer 做層次化深度檢測，GPU 端並行處理

  • 最徹底的隱藏面消除
  • 需要額外 mip-level depth mipmap
  • 適合遠景、中等密度場景

GPU-driven rendering 必備

D

Occlusion Culling（遮擋裁剪）

被其他物件完全遮擋的不渲染，遮擋物變化時重新檢測

  • 室內場景特別有效
  • 室外場景效果有限（天空遮擋少）
  • 實現複雜度中等

室內場景神器

Frustum Culling 核心原理

6 個視錐平面：

  • Near / Far plane
  • Left / Right plane
  • Top / Bottom plane

每個物件的 AABB 對每個 plane 做 Outside Test：

dot(center, plane.normal) + d [>|+radius|? → outside]

AABB 完全在任意 plane 外側 → 不可見，cull 掉

Engine 中的實作位置：

mesh_bridge_3d_create_mesh(entity) → 計算物件 world-space AABB → 存入 RenderObject3D.bounds sys_3d_render_commands() → deferred_shadow_render() → 對 visible entities 做 frustum test → deferred_gbuffer_render_geometry() → 對 visible entities 做 frustum test

每個 Pass 前 filter 一次 visible list

Hi-Z Culling 原理

Pass 1: 建立 depth mipmap（每層是前一层 1/4 大小） depth_mip_0 = main.depth (full res) depth_mip_1 = quarter(depth_mip_0) depth_mip_2 = quarter(depth_mip_1) ... Pass 2: GPU CS 查询物件包圍盒的 Hi-Z level hiz_level = log2(bounds_size_in_pixels / threshold) hi_z_depth = sample(depth_mip[hiz_level], bounds_center_uv) if (bounds_min_z > hi_z_depth) → cull Pass 3: 對通関 Hi-Z 的物件才 draw

與 Frustum Culling 的比較：

FrustumHi-Z
檢測維度3D 幾何2D Depth
處理位置CPUGPU (CS)
遮擋感知無有
額外內存無Depth mipmap
延遲無落後 1-2 幀

推薦：Frustum → Hi-Z 兩層

1
Frustum Culling (CPU) — 每幀對所有 RenderObject3D 做 AABB vs 6 planes，產生 visible list
2
Hi-Z Culling (GPU CS) — 對 visible list 中的大物件做 depth mipmap 查詢，進一步剔除被遮擋的
3
Draw Indirect — 只對最終 visible 物件發送 draw call
