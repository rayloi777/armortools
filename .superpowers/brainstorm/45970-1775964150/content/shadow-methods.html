多級陰影圖方案比較

點擊方案查看詳細說明

A

CSM (Cascaded Shadow Maps)

經典方案：多層 shadow map 覆蓋近/中/遠距離

  • 3-4 層 cascade，覆盖 camera frustum 分段
  • 近處高解析度、遠處低解析度
  • 需要選擇 cascade 的 fragment 使用哪層

成熟穩定、相容性最好

B

VSM (Variance Shadow Maps)

統計方法：每層記錄 (depth, depth²)，查詢時用 Chebyshev 估計陰影比例

  • 單層可覆蓋全距離（需 2048×2048+）
  • 支援 PCF 模糊、Biased Mipmapping
  • Light-bleeding 問題可用 Bretesta 修正

軟陰影、暈開 shadow edge

C

Ray-Traced Shadows (RTX)

硬體光線追蹤：每個 fragment 發射 ray 檢測遮擋

  • 無限解析度、完美的接觸陰影
  • 需要 DXR / Metal Ray Tracing
  • 效能成本高，適合近處物體

最高質量、需要硬體支援

D

DSM + Screen-space blur

方向陰影圖 + 後處理模糊：用 2D blur kernel 在 shadow map 上做均值擴散

  • 用 Poisson disk 或 Gaussian blur 代替 PCF
  • 減少 shadow map 解析度需求
  • blur radius 控制陰影軟硬

節省記憶體、質量中等

推薦：VSM + CSM 混合方案

推薦理由

  • VSM 只需 1-2 層 shadow map（近/遠各一），節省 draw calls
  • 可用單一 2048×2048 覆蓋近景 + 1024×1024 覆蓋遠景
  • 支援 Mipmap，可在 lighting pass 中動態選擇 LOD 等級
  • 比起 CSM 不需要 fragment 判斷使用哪層 cascade
  • Blur 成本：可預計算或在 lighting pass 同時做

缺點 / 注意

  • Light bleeding 需要用 Back-off 或 LiSBM 修正
  • 不適合點光源 spot（需 cubemap，VSM 不支援 cubemap）
  • shadow map 需要 generate mipmaps（額外 pass）
